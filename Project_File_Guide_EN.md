# Project File Guide

## Notes

- This document focuses first on maintainable project content: your own scripts, GUIs, evaluation entrypoints, and dataset folders.
- For very large repetitive file sets, this guide documents them by file pattern instead of listing every file one by one.
- The following are documented only at directory level instead of file-by-file: `llama.cpp/`, `test_GUI/node_modules/`, `result*/` output folders, `__pycache__/`, and `.idea/`.
- Some entries under `jiuge_moer/`, `jiuge_muxi/`, and `jiuge_tianshu/` are inferred from the symmetric folder structure and representative scripts.

## Root Directory

| Path | Role |
| --- | --- |
| `.idea/` | IDE project metadata, typically generated by JetBrains tools. |
| `data_sets/` | Central storage for benchmark source datasets. |
| `eval_gsm8k/` | Standalone Python implementation and results for GSM8K evaluation. |
| `grade-school-math/` | Vendored upstream GSM8K / grade-school-math project copy. |
| `jiuge_GUI/` | Electron GUI used to launch benchmark scripts from `jiuge_muxi`. |
| `jiuge_moer/` | Node.js benchmark scripts targeting the “moer” backend or port. |
| `jiuge_muxi/` | Node.js benchmark scripts targeting the “muxi” backend or port, including a connectivity test. |
| `jiuge_nvda/` | The most systematic benchmark folder, including shared helpers, V1 scripts, and summary logic. |
| `jiuge_tianshu/` | Node.js benchmark scripts targeting the “tianshu” backend or port. |
| `llama.cpp/` | Large third-party inference framework source kept as a dependency or reference. |
| `MMLUtest/` | Upstream MMLU evaluation code and original split data. |
| `test_GUI/` | A second Electron GUI, mainly used to launch scripts in `jiuge_nvda`. |
| `package-lock.json` | Root-level npm lockfile with minimal content; likely a placeholder or historical leftover. |
| `如何优化提取答案逻辑的.md` | Chinese notes document about improving answer extraction logic. |
